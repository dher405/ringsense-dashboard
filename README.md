# RingSense Call Insights Dashboard v2

A full-stack application for viewing RingCentral RingSense call insights with **server-side encrypted credential storage**, **SFTP upload**, and **scheduled automated exports**.

## Architecture

```
┌─────────────────────────┐      ┌──────────────────────────┐
│   React Frontend        │      │   Express Backend        │
│                         │ ──── │                          │
│  • Admin login          │      │  • AES-256-GCM encrypted │
│  • Settings page        │      │    credential storage    │
│  • Call list & insights │      │  • RC API proxy          │
│  • Schedule config      │      │  • SFTP upload service   │
│                         │      │  • node-cron scheduler   │
└─────────────────────────┘      └──────────────────────────┘
```

## Security

- **All secrets encrypted at rest** with AES-256-GCM using a server-side encryption key
- **Admin password** protects the dashboard UI (set via `ADMIN_PASSWORD` env var)
- **Session tokens** are time-limited (24h) and stored server-side
- **Sensitive fields never sent to browser** in plaintext — only masked values
- **httpOnly cookies** for session management
- **Helmet.js** security headers

## Features

- **Unified Settings Page** — Configure all API credentials, SFTP server, and schedule in one place
- **RingCentral JWT Auth** — Server-side token management with auto-refresh
- **Call Listing** — Browse recorded calls with search/date filtering
- **RingSense Insights** — Transcripts, summaries, highlights, next steps, AI scores
- **SFTP Upload** — Test connection and upload on demand
- **Scheduled Uploads** — Daily, weekly, or monthly automatic SFTP exports
- **Upload History** — Track scheduled job results

## Quick Start

```bash
git clone https://github.com/YOUR_USERNAME/ringsense-dashboard.git
cd ringsense-dashboard
npm install

# Generate encryption key
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"

# Create .env from template
cp .env.example .env
# Edit .env with your ENCRYPTION_KEY and ADMIN_PASSWORD

# Start dev servers (backend + frontend)
npm run dev
```

Frontend: http://localhost:3000 → Backend: http://localhost:4000

## Deploy to Render

1. Push to GitHub
2. Create a new **Web Service** (not Static Site) on Render
3. Connect the repo
4. Set environment variables in Render dashboard:
   - `ENCRYPTION_KEY` — 64-char hex string (auto-generated by render.yaml)
   - `SESSION_SECRET` — random string (auto-generated)
   - `ADMIN_PASSWORD` — your chosen admin password
   - `NODE_ENV` — `production`
5. Build command: `npm install && npm run build`
6. Start command: `npm start`

## Settings Page Sections

### RingCentral API
- Server URL (production vs sandbox)
- Client ID & Client Secret
- JWT Credential
- Test Connection button

### SFTP Server
- Host, port, username
- Password or SSH key authentication
- Remote upload path
- Test Connection & Upload Now buttons

### Scheduled Uploads
- Enable/disable toggle
- Frequency: daily, weekly, or monthly
- Time of day & timezone
- Day of week (weekly) or day of month (monthly)
- Lookback period (how many days of data to include)
- Run Now button
- Upload history log
